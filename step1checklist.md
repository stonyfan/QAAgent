# QA Agent 第一阶段完成标准（API测试闭环）

---

## 🎯 阶段目标

构建一个完整的 API 自动化测试闭环：

输入 API（描述 / Swagger） → 自动生成测试 → 自动执行 → 自动输出报告

---

## ✅ 一、功能完成标准

### 1️⃣ Swagger解析（可选但建议完成）

* [ ] 能读取本地 swagger.json
* [ ] 能解析 paths / method / parameters
* [ ] 能提取多个 API
* [ ] 能转换为标准 API 描述（字符串）

---

### 2️⃣ API测试生成（核心）

* [ ] 能根据 API 描述生成 pytest 代码
* [ ] 每个 API 生成独立 test_xxx.py 文件
* [ ] 至少包含：

  * [ ] 正常测试
  * [ ] 异常测试
* [ ] 包含 assert（状态码 + 内容）
* [ ] 代码可直接运行（无语法错误）

---

### 3️⃣ 测试执行（test_runner）

* [ ] 能调用 pytest（subprocess）
* [ ] 能执行 tests/ 目录所有测试
* [ ] 能返回完整执行结果（stdout）

---

### 4️⃣ 报告生成（report）

* [ ] 能解析 pytest 输出
* [ ] 能统计：

  * [ ] 通过数量
  * [ ] 失败数量
* [ ] 能列出失败测试
* [ ] 能生成文本报告

---

### 5️⃣ 主控流程（Agent / Pipeline）

* [ ] 能串联：

  * swagger_parser
  * api_test
  * test_runner
  * report
* [ ] 一条命令执行完整流程
* [ ] 输出最终报告

---

## 🧪 二、可运行标准（必须满足）

* [ ] 整个流程可以一键运行
* [ ] 不需要手动修改代码
* [ ] 无崩溃（允许测试失败，但流程不能崩）

---

## 📂 三、文件结构标准

* [ ] tests/ 目录自动生成测试文件
* [ ] 每个 API 对应一个文件
* [ ] 文件命名清晰（test_login.py）

---

## 🧠 四、代码质量标准

* [ ] 每个模块独立
* [ ] 函数职责单一
* [ ] 不跨模块耦合
* [ ] 无明显冗余代码

---

## 🚫 五、常见失败判定（踩坑警告）

以下任一情况 = 第一阶段未完成：

* [ ] 只能生成代码但不能运行
* [ ] 只能测一个API
* [ ] 无法批量处理 Swagger
* [ ] 需要人工频繁修改代码
* [ ] 没有测试报告

---

## 🎯 完成标志（最终验证）

你可以做到：

输入：

“使用这个 swagger.json 生成测试并执行”

输出：

✔ 自动生成多个测试文件
✔ 自动执行 pytest
✔ 自动输出测试报告

---

## 🏁 阶段结论

如果以上全部完成：

👉 第一阶段（API测试闭环）完成 ✅
👉 已具备真实项目使用能力 ✅
