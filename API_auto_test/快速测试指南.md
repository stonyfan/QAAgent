## 🔍 查看生成的测试文件

```bash
# 列出所有生成的测试文件
ls -la tests/

# 查看某个测试文件的内容
cat tests/test_get_posts.py
```

**示例测试文件：**
```python
import requests
import pytest

# BASE_URL 从 Swagger 文档的 servers 字段自动提取
BASE_URL = "https://jsonplaceholder.typicode.com"

def test_posts_success():
    """测试 posts 正常场景"""
    url = f"{BASE_URL}/posts"
    method = "GET"

    response = requests.get(url)

    assert response.status_code == 200
    assert "data" in response.json() or "message" in response.json()

def test_posts_invalid_params():
    """测试 posts 异常场景 - 缺少参数"""
    url = f"{BASE_URL}/posts"
    method = "GET"

    response = requests.get(url)  # 缺少查询参数

    assert response.status_code in [400, 422]
```

## 🛠️ 手动运行测试（可选）

如果你想单独运行某个测试：

```bash
# 运行所有测试
pytest tests/ -v

# 运行特定测试文件
pytest tests/test_get_posts.py -v

# 运行特定测试函数
pytest tests/test_get_posts.py::test_posts_success -v

# 显示详细输出
pytest tests/ -v -s
```

## ❓ 常见问题

### Q: 为什么有些测试会失败？
A: JSONPlaceholder是模拟API，POST/PUT操作不会真实保存数据，但会返回成功响应。

### Q: 如何测试自己的API？
A:
1. 创建你的Swagger配置文件，在 `servers` 字段中指定你的API地址
2. 运行测试：`python src/pipeline.py your-swagger.json`

### Q: 如何修改BASE_URL？
A: 编辑 swagger.json 的 `servers` 字段，然后重新生成测试。详见：[BASE_URL配置指南.md](BASE_URL配置指南.md)

### Q: 可以同时测试多个API吗？
A: 可以！创建包含多个API的Swagger配置文件即可。
