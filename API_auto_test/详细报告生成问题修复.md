# 详细报告生成问题修复

## 🐛 问题描述

运行测试后，详细报告没有生成，提示：
```
⚠ 没有找到详细日志，跳过详细报告生成
```

---

## 🔍 问题原因

**路径不匹配：**

- 测试代码将日志写入：`reports/api_detailed_log.jsonl`
- 报告生成函数在子目录查找：`reports/20260426_155152/api_detailed_log.jsonl`

**为什么会这样？**

1. 测试在 `report_dir` 创建**之前**运行
2. 所以日志文件写入到默认的 `reports/` 根目录
3. 但报告生成函数只在 `report_dir` 子目录中查找

---

## ✅ 修复方案

修改了 `src/report.py` 中的 `generate_and_save_detailed_report()` 函数：

**修复前：**
```python
# 只在指定的output_dir中查找
log_file = os.path.join(output_dir, 'api_detailed_log.jsonl')
if not os.path.exists(log_file):
    return None
```

**修复后：**
```python
# 先在output_dir中查找
log_file = os.path.join(output_dir, 'api_detailed_log.jsonl')

# 如果找不到，在reports根目录查找
if not os.path.exists(log_file):
    if output_dir.startswith('reports/'):
        reports_root = 'reports'
        log_file = os.path.join(reports_root, 'api_detailed_log.jsonl')

if not os.path.exists(log_file):
    return None
```

```
2. 创建报告目录

   └─> reports/20260426_155152/API测试报告.txt

4. 生成详细报告
   ├─> 在 reports/20260426_155152/ 中查找日志（未找到）
   ├─> 在 reports/ 中查找日志（找到！）✓
   └─> reports/20260426_155152/API测试详细报告.txt

5. 清理日志文件
   └─> 删除 reports/api_detailed_log.jsonl
```
