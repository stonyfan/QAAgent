## 📦 安装步骤

### 1️⃣ 系统要求

- Python 3.7 或更高版本
- pip（Python包管理器）

### 2️⃣ 安装依赖

```bash
# 安装必需的包
pip install pytest requests
```

**依赖说明：**
- `pytest`: Python测试框架
- `requests`: HTTP请求库

### 3️⃣ 验证安装

```bash
# 检查pytest是否安装成功
pytest --version

# 检查requests是否安装成功
python -c "import requests; print(requests.__version__)"
```

## 📄 准备Swagger文档

### Swagger文档格式要求

支持 **OpenAPI 3.0** 和 **Swagger 2.0** 格式。

### 示例：最小化Swagger文档

```json
{
  "openapi": "3.0.0",
  "info": {
    "title": "示例API",
    "version": "1.0.0"
  },
  "paths": {
    "/api/login": {
      "post": {
        "summary": "用户登录",
        "responses": {
          "200": {
            "description": "成功"
          }
        }
      }
    }
  }
}
```

### 获取Swagger文档的方式

#### 方式1: 使用项目自带的示例
```bash
# 项目已包含 swagger.json 示例
python src/pipeline.py swagger.json
```

#### 方式2: 从在线API下载
```bash
# 使用curl下载Swagger文档
curl https://api.example.com/swagger.json -o my-api.json

# 运行测试
python src/pipeline.py my-api.json
```

#### 方式3: 手动创建
创建 `my-api.json` 文件，按照上述格式填写API信息。

## 🔧 高级用法

### 1️⃣ 单独使用各个模块

#### 只解析Swagger文档

```python
from src.swagger_parser import parse_swagger

apis = parse_swagger('swagger.json')
for api in apis:
    print(f"{api['method']} {api['path']}")
```

#### 只生成测试代码

```python
from src.swagger_parser import parse_swagger
from src.api_test import generate_tests_for_apis

apis = parse_swagger('swagger.json')
test_files = generate_tests_for_apis(apis, output_dir='tests')
print(f"生成了 {len(test_files)} 个测试文件")
```

#### 只运行测试

```python
from src.test_runner import run_pytest

results = run_pytest('tests')
print(f"通过: {results['passed']}, 失败: {results['failed']}")
```

#### 只生成报告

```python
from src.report import generate_and_save_report
from src.test_runner import run_pytest

results = run_pytest('tests')
report_path = generate_and_save_report(
    results,
    source="我的API测试",
    output_dir='reports'
)
print(f"报告: {report_path}")
```

### 2️⃣ 自定义测试配置

#### 修改Base URL

编辑生成的测试文件，修改 `BASE_URL`:

```python
# tests/test_post_login.py
BASE_URL = "https://api.example.com"  # 改为实际API地址
```

#### 添加认证信息

```python
# 在测试函数中添加headers
def test_login_success():
    headers = {
        'Authorization': 'Bearer your-token-here'
    }
    response = requests.post(url, headers=headers, json={...})
```

### 3️⃣ 集成到CI/CD

#### GitHub Actions 示例

```yaml
name: API Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python

          python-version: '3.8'

        run: pip install pytest requests
      - name: Run API tests
        run: python src/pipeline.py swagger.json
      - name: Upload reports
        uses: actions/upload-artifact@v2
        with:
          name: test-reports
```

## ❓ 常见问题

### Q1: 提示 "pytest: command not found"

**解决方法:**
```bash
pip install pytest
```

---

### Q2: 所有测试都失败了

**可能原因:**
1. API服务未启动
2. BASE_URL配置错误
3. API接口已变更

**解决方法:**
1. 确认API服务正在运行
2. 修改测试文件中的 `BASE_URL`
3. 检查API文档是否更新

---

### Q3: 某个测试失败了怎么办？

**步骤:**
1. 查看测试报告中的失败详情
2. 打开对应的测试文件
3. 检查断言是否合理
4. 手动测试API验证

---

### Q4: 如何只运行某个测试？

```bash
# 运行单个测试文件
pytest tests/test_post_login.py

# 运行特定的测试函数
pytest tests/test_post_login.py::test_login_success

# 运行匹配名称的测试
pytest tests/ -k login
```

---

### Q5: 生成的测试代码太简单怎么办？

当前版本使用模板生成基础测试。未来版本将集成LLM生成更智能的测试。

**临时解决方案:**
手动编辑生成的测试文件，添加更多测试场景。
