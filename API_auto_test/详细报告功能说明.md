# 详细报告功能说明

## 📋 功能概述

QA Agent 现在支持生成两种测试报告：
1. **API测试报告** - 摘要报告（原有）
2. **API测试详细报告** - 详细报告（新增）

---

## 🎯 报告结构

```
reports/
└── 20260426_153836/              # 以日期时间命名的文件夹
    ├── API测试报告.txt            # 摘要报告
    └── API测试详细报告.txt        # 详细报告
```

> 💡 **提示：** 每次测试运行会创建一个新文件夹，文件名固定，方便查找和对比。详见：[报告结构调整.md](报告结构调整.md)

## 📊 报告对比

### 摘要报告（api_test_report_*.txt）

**用途：** 快速了解测试整体情况

**内容：**
- ⚠️ 风险评估

- ✅ 通过的测试列表
- ❌ 失败的测试详情（如有）
- 📄 完整pytest输出

**示例：**

### 详细报告（api_test_detailed_report_*.txt）

**用途：** 深入分析每个API请求和响应

**内容：**
- 📤 每个测试的请求信息
  - HTTP方法
  - 完整URL
  - 请求体（如有）

- 📥 每个测试的响应信息
  - 状态码
  - 响应时间（毫秒）
  - 响应头
  - 响应体

**示例：**

## 🔧 工作原理

### 1. 日志记录

测试代码自动记录每个API调用：

```python
def log_request_response(test_name, url, method, response, request_body=None):
    """记录API请求和响应的详细信息"""
    log_entry = {
        "timestamp": datetime.now().isoformat(),
        "test_name": test_name,
        "request": {
            "url": url,
            "method": method,
            "body": request_body
        },
        "response": {
            "status_code": response.status_code,
            "headers": dict(response.headers),
            "body": response.text[:1000],  # 限制大小
            "response_time_ms": response.elapsed.total_seconds() * 1000
        }
    }

    # 写入 JSON Lines 格式日志
    with open('reports/api_detailed_log.jsonl', 'a') as f:
        f.write(json.dumps(log_entry) + '\n')
```

### 2. 报告生成

测试完成后，系统自动：
1. 读取 `api_detailed_log.jsonl`
2. 解析所有日志条目
3. 生成易读的详细报告
4. 清理临时日志文件

## 📈 详细报告的价值

### 1️⃣ 性能分析

**发现慢API：**
```
测试: test_get_posts_success
  响应时间: 1245.67 ms  ← 比较慢

测试: test_get_users_success
  响应时间: 89.23 ms   ← 正常
```

### 2️⃣ 数据验证

**检查返回数据：**
```
响应体:
{
  "userId": 1,
  "id": 1,
  "title": "...",
  "body": "..."
}
```

确认API返回的数据结构符合预期。

### 3️⃣ 问题诊断

**定位失败原因：**
```
状态码: 401
响应体:
{
  "error": "Unauthorized",
  "message": "Invalid token"
}
```

快速识别认证问题。

### 4️⃣ API文档

**实际响应示例：**
详细报告可以作为API文档的补充，展示真实的请求和响应。

## 🔍 报告内容详解

### 请求信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 方法 | HTTP方法 | GET, POST, PUT, DELETE |
| URL | 完整请求URL | https://api.example.com/posts |
| 请求体 | POST/PUT的请求体 | {"title": "test"} |

### 响应信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 状态码 | HTTP状态码 | 200, 201, 404, 500 |
| 响应时间 | 请求耗时（毫秒） | 245.67 ms |
| 响应头 | HTTP响应头 | content-type, content-length等 |
| 响应体 | 响应内容（限制1000字符） | JSON或文本 |

## 🛠️ 自定义配置

### 修改响应体大小限制

在 `api_test.py` 中：

```python
# 默认限制1000字符
"body": response.text[:1000]

# 改为2000字符
"body": response.text[:2000]
```

### 调整日志保留

在 `report.py` 中：

```python
# 默认：生成报告后删除日志文件
os.remove(log_file)

# 改为保留日志文件
# os.remove(log_file)  # 注释掉这行
```

## ❓ 常见问题

### Q: 详细报告太大怎么办？

**A:** 可以限制响应体大小或减少记录的测试数量。

### Q: 能否只生成详细报告？

**A:** 可以，修改 `pipeline.py` 跳过摘要报告生成。

### Q: 日志文件在哪里？

**A:** `reports/api_detailed_log.jsonl`，生成报告后会自动删除。

### Q: 能否导出为JSON格式？

**A:** 可以修改 `generate_detailed_report()` 函数输出JSON格式。
