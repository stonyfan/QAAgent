## 🎯 异常测试类型

### 根据HTTP方法生成的异常测试

| HTTP方法 | 异常测试类型 | 测试场景 | 预期结果 |
|---------|-----------|---------|---------|
| **GET** | 资源不存在 | 使用不存在的ID（999999） | 返回404 |
| **GET** | 无效参数（列表API） | 发送无效查询参数 | 返回200/400/422 |
| **POST** | 缺少必需字段 | 发送空请求体 `{}` | 返回400或422 |
| **PUT** | 更新不存在资源 | 使用不存在的ID（999999） | 返回404 |
| **DELETE** | 删除不存在资源 | 使用不存在的ID（999999） | 返回404 |

## 📝 生成的测试示例

### GET请求（详情API）

```python
def test_get_post_detail_success():
    """测试 get_post_detail 正常场景"""
    url = BASE_URL + "/posts/1"
    response = requests.get(url)
    assert response.status_code == 200


def test_get_post_detail_not_found():
    """测试 get_post_detail 资源不存在场景"""
    url = BASE_URL + "/posts/999999"  # 不存在的ID
    response = requests.get(url)
    assert response.status_code == 404, "不存在的资源应返回404"
```

### POST请求

```python
def test_post_post_success():
    """测试 post_post 正常场景"""
    url = BASE_URL + "/posts"
    response = requests.post(url, json={"key": "value"})
    assert response.status_code in [200, 201]


def test_post_post_missing_fields():
    """测试 post_post 缺少必需字段场景"""
    url = BASE_URL + "/posts"
    response = requests.post(url, json={})  # 空请求体
    assert response.status_code in [400, 422], "缺少必需字段应返回400或422"
```

### DELETE请求

```python
def test_delete_post_detail_success():
    """测试 delete_post_detail 正常场景"""
    url = BASE_URL + "/posts/1"
    response = requests.delete(url)
    assert response.status_code in [200, 204]


def test_delete_post_detail_not_found():
    """测试 delete_post_detail 删除不存在的资源"""
    url = BASE_URL + "/posts/999999"  # 不存在的ID
    response = requests.delete(url)
    assert response.status_code == 404, "删除不存在的资源应返回404"
```

## 🔍 异常测试覆盖的场景

### 1️⃣ 资源不存在（404错误）

**适用API：** GET/PUT/DELETE详情API

**测试方法：** 使用不存在的资源ID（999999）

**验证：** API返回404状态码

```python
url = BASE_URL + "/posts/999999"
response = requests.get(url)
assert response.status_code == 404
```

---

### 2️⃣ 缺少必需字段（400/422错误）

**适用API：** POST/PUT创建API

**测试方法：** 发送空请求体

**验证：** API返回400或422状态码

```python
url = BASE_URL + "/posts"
response = requests.post(url, json={})
assert response.status_code in [400, 422]
```

---

### 3️⃣ 无效查询参数（容错测试）

**适用API：** GET列表API

**测试方法：** 发送无效查询参数

**验证：** API能正常处理（200或错误码）

```python
url = BASE_URL + "/posts?invalid_param=test"
response = requests.get(url)
assert response.status_code in [200, 400, 422]
```

## 💡 异常测试的价值

### 1️⃣ 验证API的错误处理

确保API在异常情况下能正确返回错误码，而不是崩溃。

### 2️⃣ 提高API健壮性

发现API在边界情况下的行为问题。

### 3️⃣ 文档化错误响应

明确记录API在不同错误情况下的预期行为。

### 4️⃣ 防止生产事故

提前发现可能导致生产问题的错误处理缺陷。
