# QA Agent 快速参考

## 🚀 快速开始

```bash
# 安装依赖
pip install pytest requests

# 运行测试
python src/pipeline.py swagger.json

# 查看报告
cat reports/api_test_report_*.txt
```

## 📁 项目结构

```
QA agent/
├── src/
│   ├── swagger_parser.py   # Swagger解析
│   ├── api_test.py          # 测试生成
│   ├── test_runner.py       # 测试执行
│   ├── report.py            # 报告生成
│   └── pipeline.py          # 主控流水线
├── tests/                   # 测试文件
├── reports/                 # 测试报告
└── swagger.json             # 示例文档
```

---

## 💡 常用命令

### 基本使用
```bash
# 使用自定义Swagger文档
python src/pipeline.py path/to/api.json

# 只运行某个测试
pytest tests/test_post_login.py

# 运行所有测试
pytest tests/ -v
```

### 查看结果
```bash
# 查看最新报告
cat reports/api_test_report_$(ls -t reports/ | head -1)

# 列出所有报告
ls -lh reports/

# 查看生成的测试文件
ls -lh tests/
```

headers = {'Authorization': 'Bearer token'}

## 📊 报告说明

### 风险等级
- 🟢 **LOW** - 所有测试通过
- 🟡 **MEDIUM** - 少量测试失败（<20%）
- 🟠 **HIGH** - 较多测试失败（20-50%）
- 🔴 **CRITICAL** - 大部分测试失败（>50%）

### 测试文件命名
- `test_post_login.py` - POST /api/login
- `test_get_users.py` - GET /api/users
- `test_get_user.py` - GET /api/users/{id}

## ❓ 常见问题

| 问题 | 解决方法 |
|------|---------|
| pytest未找到 | `pip install pytest` |
| 所有测试失败 | 检查BASE_URL和API服务状态 |
| 部分测试失败 | 查看报告中的失败详情 |
