# 报告文件结构调整说明

## 📋 新的文件结构

### 旧结构
```
reports/
├── api_test_report_20260426_154515.txt
└── api_test_detailed_report_20260426_154515.txt
```

### 新结构 ✅
```
reports/
└── 20260426_154515/                    # 以日期时间命名的文件夹
    ├── API测试报告.txt                  # 摘要报告
    └── API测试详细报告.txt              # 详细报告
```

---

## 🎯 改进优势

| 特性 | 旧结构 | 新结构 |
|------|--------|--------|
| **组织性** | 所有报告混在一起 | 按时间分组，清晰明了 |
| **查找** | 需要记住文件名时间戳 | 直接找到对应时间文件夹 |
| **对比** | 需要复制粘贴文件名 | 直接对比不同文件夹 |
| **归档** | 需要手动整理 | 天然按时间归档 |
| **文件名** | 包含时间戳，较长 | 固定名称，简洁 |

## 🛠️ 实现细节

### pipeline.py 修改
```python
# 创建报告目录（以时间戳命名）
timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
report_dir = os.path.join('reports', timestamp)
os.makedirs(report_dir, exist_ok=True)

# 传递给报告生成函数
report_path = generate_and_save_report(
    test_results,
    source=f"Swagger: {swagger_path}",
    output_dir=report_dir  # 使用新的目录
)
```

### report.py 修改

**摘要报告：**
```python
# 固定文件名
report_filename = 'API测试报告.txt'
report_path = os.path.join(output_dir, report_filename)
```

**详细报告：**
```python
# 固定文件名
report_filename = 'API测试详细报告.txt'
report_path = os.path.join(output_dir, report_filename)
```

## ❓ 常见问题

### Q: 如何找到特定时间段的报告？
**A:** 
```bash
# 列出所有时间文件夹
ls reports/

# 查找今天下午的报告
ls reports/20260426_1*/
```

### Q: 可以自定义时间格式吗？
**A:** 可以，修改 `pipeline.py` 中的时间戳格式：
```python
# 改为更易读的格式
timestamp = datetime.now().strftime('%Y-%m-%d_%H-%M-%S')
# 结果: 2026-04-26_15-45-15/
```

### Q: 能否在同一时间创建多次报告？
**A:** 不建议。如果时间戳相同，新的报告会覆盖旧的。建议至少间隔1分钟。

### Q: 如何导出某次测试的所有报告？
**A:** 
```bash
# 导出特定时间的报告
cp -r reports/20260426_154515 /path/to/destination/
```
