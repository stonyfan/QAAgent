# QA Agent 项目说明（强化版）

## 🎯 项目目标

构建一个模块化 QA Agent，用于通过自然语言驱动 QA 工作流。

系统特点：

- 主控（Agent） + 多模块（工具）
- 每个模块独立运行
- 支持逐步执行（非全自动流水线）
- 优先实现 API 测试闭环

---

## 🏗️ 架构设计

### 1. 主控 Agent

职责：

- 解析用户输入
- 路由到对应模块
- 不做复杂业务逻辑
- 不生成测试内容

---

### 2. 模块划分（必须独立）

每个模块必须：

- 独立函数实现
- 不直接依赖其他模块
- 可单独运行测试

模块包括：

- flow：测试流程生成
- strategy：测试策略生成
- testcase：测试用例生成
- api_test：API测试生成与执行
- report：测试报告生成

---

## ⚙️ 开发原则（必须遵守）

1. 一次只实现一个模块
2. 每一步必须可运行
3. 不允许跨模块开发
4. 先设计，再实现
5. 每个函数必须职责单一
6. 输出必须结构清晰

---

## 🚫 禁止事项（非常重要）

- ❌ 不要一次实现多个模块
- ❌ 不要自动扩展功能
- ❌ 不要引入复杂框架（如LangChain）
- ❌ 不要生成无法运行的代码
- ❌ 不要跳过设计直接写代码

---

## 🧪 开发流程（必须按顺序）

1. 模块设计
2. Prompt设计
3. 代码实现
4. 本地运行
5. 错误修复
6. 优化

---

## 🧰 技术栈

- Python
- pytest
- requests
